# Vérifier une réponse IA

<div className="coursModuleMeta">
  <span>20–30 min</span>
  <span>Notion transversale</span>
  <span>Validation</span>
</div>

<p className="coursModuleLead">
Une réponse IA peut être claire, élégante et fausse. Dans cette rencontre, on apprend à vérifier une réponse avant de l’accepter, de la remettre ou de l’intégrer dans un projet.
</p>

## ✅ Plausible ne veut pas dire vérifié

Voici une mauvaise habitude très tentante :

<div className="aiBox aiBoxPrompt aiBoxSmall">
  <div className="aiBoxHeader">
    <span className="aiBoxTitle">Réflexe dangereux</span>
    <span className="aiBoxMeta">À éviter</span>
  </div>
  <div className="aiBoxBody">
    <pre><code>{`La réponse a l’air bonne, donc je la prends.`}</code></pre>
  </div>
</div>

Le problème, c’est que :

- l’IA peut avoir ignoré une contrainte;
- le code peut fonctionner seulement dans un cas simple;
- l’explication peut être plausible mais incorrecte;
- la solution peut contourner le vrai problème;
- tu peux être incapable de justifier le résultat.

:::danger
Plausible ne veut pas dire vérifié. C’est peut-être la phrase la plus utile du cours.
:::

## 🔍 Ce qu’il faut vérifier

Valider une réponse IA, ce n’est pas seulement relire rapidement.

C’est vérifier :

1. la correspondance avec la demande;
2. le respect des contraintes;
3. la présence de cas limites;
4. la cohérence avec le projet;
5. la capacité à expliquer la solution;
6. la possibilité de tester le résultat.

Une bonne validation transforme une impression en décision.

## 🤖 Exemple de réponse trop convaincante

Une réponse peut sembler solide même si elle oublie une contrainte.

<div className="aiBox aiBoxResponse">
  <div className="aiBoxHeader">
    <span className="aiBoxTitle">Réponse de l’IA</span>
    <span className="aiBoxMeta">Plausible, mais à vérifier</span>
  </div>
  <div className="aiBoxBody">
    <p>J’ai ajouté la validation demandée. Le formulaire devrait maintenant empêcher les valeurs invalides et afficher un message d’erreur.</p>
  </div>
</div>

Cette réponse est rassurante, mais elle ne prouve pas que :

- la bonne règle a été appliquée;
- tous les cas importants sont couverts;
- le message est au bon endroit;
- le projet compile encore;
- le comportement a été testé.

## 🧰 Checklist de validation

Avant d’accepter une réponse, pose-toi ces questions :

- [ ] La réponse répond à la demande réelle.
- [ ] Les contraintes importantes sont respectées.
- [ ] Les cas limites évidents sont considérés.
- [ ] La solution est cohérente avec le projet.
- [ ] Je peux expliquer la solution dans mes mots.
- [ ] Je peux vérifier le résultat avec un test, une exécution ou une inspection.

## 🧪 À essayer

Lis une réponse IA volontairement plausible mais incomplète.

Ta tâche :

- identifier ce qui semble correct;
- identifier au moins deux risques;
- écrire une question de clarification;
- proposer une vérification ou un test;
- décider : accepter, corriger, rejeter ou demander une autre passe.

## 🔩 Récapitulatif

- Plausible ne veut pas dire correct.
- Une réponse IA doit être vérifiée comme une contribution externe.
- Une checklist courte vaut mieux qu’une confiance vague.
- La personne garde les mains sur le volant.

## 🚀 Pour aller plus loin

- **Essentiel** : appliquer une checklist de validation à une réponse IA courte.
- **Consolidation** : écrire un test, une vérification ou une question de clarification.
- **Défi** : comparer plusieurs réponses IA avec des critères explicites.
