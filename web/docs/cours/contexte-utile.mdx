# Donner un contexte utile à l’IA

<div className="coursModuleMeta">
  <span>20–30 min</span>
  <span>GitHub Copilot</span>
  <span>Programmation</span>
</div>

<p className="coursModuleLead">
Quand on demande simplement “Corrige mon code”, l’IA doit deviner beaucoup trop de choses. Dans ce module, on apprend à transformer une demande floue en demande claire, contextualisée et vérifiable.
</p>

## Le problème

Voici une demande typique, mais fragile :

```text
Corrige mon code, ça ne marche pas.
```

Cette demande force l’IA à deviner :

- quel comportement est attendu;
- quel fichier regarder;
- quelle erreur apparaît;
- ce qui a déjà été essayé;
- ce qui compte comme une bonne solution.

:::warning
Quand l’IA manque de contexte, elle ne s’arrête pas toujours. Elle peut produire une réponse plausible, mais fragile.
:::

## Ce qu’il faut comprendre

Un bon contexte répond habituellement à cinq questions :

1. Quelle est la tâche exacte?
2. Quel est le résultat attendu?
3. Quels fichiers ou extraits sont pertinents?
4. Quelles contraintes doivent être respectées?
5. Comment saura-t-on que c’est réussi?

Le contexte doit être assez riche pour orienter l’IA, mais assez court pour rester contrôlable.

## Exemple guidé

On part d’une demande floue :

```text
Ajoute la validation du formulaire.
```

On peut la transformer en demande contextualisée :

```text
Dans le formulaire de création d’emploi, ajoute une validation côté serveur pour le champ Salaire.

Contexte :
- le formulaire existe déjà;
- le champ Nom est déjà obligatoire;
- le champ Salaire doit être obligatoire et numérique;
- on veut un message d’erreur clair en français;
- ne change pas la structure générale du formulaire.

Critère de réussite :
- si Salaire est vide, afficher “Le salaire est obligatoire”;
- si Salaire n’est pas numérique, afficher “Le salaire doit être un nombre”.
```

À observer :

- la réponse est plus précise;
- les hypothèses inutiles diminuent;
- le résultat devient plus facile à vérifier;
- la discussion avec l’IA part dans une direction plus utile.

## À toi de jouer

Transforme ce prompt flou :

```text
Ajoute la validation du formulaire.
```

en demande contextualisée.

Ta version doit inclure :

- la fonctionnalité attendue;
- le fichier ou composant concerné;
- les règles de validation;
- le comportement attendu en cas d’erreur;
- un critère de réussite.

## À retenir

- Un prompt flou force l’IA à deviner.
- Un bon contexte réduit les hypothèses inutiles.
- Le contexte doit servir la tâche, pas raconter toute l’histoire du projet.
- Les critères de réussite sont aussi importants que la demande.

## Pour aller plus loin

- **Essentiel** : comparer un prompt flou et un prompt contextualisé.
- **Consolidation** : choisir quelles informations inclure ou exclure.
- **Défi** : gérer un contexte multi-fichiers ou une session longue.
