# Modules pilotes

> Source : `project/current/cours-ia-avance/modules/`  
> Identifiant : `[DOC-PAR-003]`  
> Version : 0.1  
> Dernière modification : 2026-06-12  
> Statut : index de travail

Cette page présente les trois premiers micro-modules pilotes du parcours **Coder avec l’IA**.

Ils servent à valider les gabarits pédagogiques avant de produire l’ensemble du cours.

## Modules pilotes

| ID | Module | Domaine | Statut | Rôle |
|---|---|---|---|---|
| `[MIC-CTX-001]` | Donner un contexte utile à l’IA | CTX | pilote | Améliorer la qualité des réponses IA |
| `[MIC-MAP-001]` | Cartographier un codebase | MAP | pilote | Trouver où intervenir dans un projet |
| `[MIC-EVA-001]` | Vérifier une réponse IA plausible | EVA | pilote | Valider avant d’accepter une réponse |

## Ce qu’on valide avec ces modules

- Le format de micro-module de 20 à 30 minutes.
- Le découpage en paliers I / II / III.
- La présence d’une friction de départ.
- La clarté du “What’s in it for me?”.
- La place de GitHub Copilot comme outil principal.
- Les équivalences Codex et Claude Code, seulement quand utiles.
- La possibilité de produire rapidement d’autres modules avec le même gabarit.

## Résumé des pilotes

### [MIC-CTX-001] Donner un contexte utile à l’IA

Objectif : apprendre à fournir à l’IA un contexte suffisant, ciblé et utile sans l’inonder d’informations inutiles.

Friction de départ : “Corrige mon code, ça ne marche pas.”

### [MIC-MAP-001] Cartographier un codebase

Objectif : comprendre où intervenir dans un projet existant avant de demander à l’IA de modifier du code.

Friction de départ : “Ajoute cette fonctionnalité dans le projet.”

### [MIC-EVA-001] Vérifier une réponse IA plausible

Objectif : valider une réponse IA qui semble convaincante avant de l’accepter, de la remettre ou de l’intégrer.

Friction de départ : “La réponse a l’air bonne, donc je la prends.”
