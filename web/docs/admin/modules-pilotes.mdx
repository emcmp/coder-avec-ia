## Comment lire les trois pilotes

| Élément | Rôle | Exemple |
|---|---|---|
| Rencontre | Position actuelle dans le parcours public | Rencontre 1 - Contexte utile |
| Page publique | Version lisible par l’étudiant | `/cours/contexte-utile` |
| Micro-module | Contenu pédagogique stable | `MIC-CTX-001` |
| Domaine | Famille conceptuelle | `CTX` |
| Compétence | Capacité observable liée au contenu | `CMP-CTX-001` |
| Source projet | Fichier de conception et de maintenance | `project/current/cours-ia-avance/modules/...` |

:::info
La rencontre donne l’ordre de présentation. Le micro-module donne l’identité stable du contenu.
:::

## Ce qu’on valide avec ces modules

- Le format de rencontre publique.
- Le découplage entre contenu stable et ordre de présentation.
- La présence d’un problème de départ reconnaissable.
- L’usage de boîtes IA pour les prompts et réponses.
- La lisibilité des références pédagogiques repliées.
- Le lien explicite entre modules et compétences observables.
- L’usage de GitHub Copilot comme exemple principal actuel, sans en faire la nature de la notion.
