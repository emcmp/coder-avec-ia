# Compétences visées

> Source : `project/catalogue/competences.md`  
> Identifiant : `[DOC-CAT-004]`  
> Version : 0.1  
> Dernière modification : 2026-06-12  
> Statut : brouillon structurant

Cette page présente les compétences visées par le parcours **Coder avec l’IA** et leur lien avec les micro-modules.

Une compétence décrit ce que la personne doit être capable de faire. Elle est transversale : une même compétence peut être développée par plusieurs modules, et un module peut contribuer à plusieurs compétences.

## Format

Les compétences utilisent le format :

```text
[CMP-DOMAINE-NNN]
```

## Compétences prioritaires pour une première version courte

| ID | Compétence | Domaine |
|---|---|---|
| `[CMP-IA-001]` | Choisir le bon type d’outil IA pour une tâche | IA |
| `[CMP-CTX-001]` | Fournir un contexte utile à un assistant IA | CTX |
| `[CMP-SPE-001]` | Spécifier une tâche avant de l’implémenter avec l’IA | SPE |
| `[CMP-MAP-001]` | Identifier où intervenir dans un codebase existant | MAP |
| `[CMP-VCS-001]` | Intégrer l’IA dans un flux Git révisable | VCS |
| `[CMP-EVA-001]` | Valider une réponse ou une production IA | EVA |
| `[CMP-SEC-001]` | Encadrer les risques, permissions et données sensibles | SEC |
