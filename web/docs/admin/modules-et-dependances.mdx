# Modules et dépendances

> Source : `project/catalogue/modules-et-dependances.md`

Cette page présente la carte canonique des micro-modules du parcours **Coder avec l’IA**.

Elle ne constitue pas encore une table des matières finale. Elle sert à suivre les identités stables, les dépendances, les statuts et les sources détaillées.

## Comment lire la carte

| Information | Source de vérité |
|---|---|
| ID, titre stable, domaine, statut et dépendances | catalogue des modules |
| Contenu pédagogique détaillé | fichier du module dans `project/current/cours-ia-avance/modules/` |
| Compétences | [Compétences visées](./competences) |
| Page étudiant | section [Cours](../cours) |
| Ordre des rencontres | index public et navigation |

## Règle provider

Les trois pilotes sont des notions transversales. GitHub Copilot est l’exemple principal actuel, mais il ne définit pas leur identité.

Un provider devient visible dans le titre ou les métadonnées seulement lorsque le contenu dépend réellement de cet outil.
