# Gabarits pédagogiques

> Source : `project/conventions/gabarits-pedagogiques.md`  
> Identifiant : `[DOC-CHR-005]`  
> Version : 0.3  
> Dernière modification : 2026-06-12  
> Statut : convention de travail initiale

Cette page présente les gabarits réutilisables pour produire les contenus pédagogiques du parcours **Coder avec l’IA**.

## Deux formats à distinguer

| Format | Rôle |
|---|---|
| Micro-module source/Admin | Concevoir, versionner et relier les contenus |
| Rencontre publique | Guider une activité lisible pour l’étudiant |

:::warning
Une page publique ne doit pas ressembler à une fiche de planification. Les identifiants, statuts et compétences restent disponibles, mais discrets.
:::

## Standard des rencontres publiques

Structure recommandée :

- titre `Cours N - Titre de la rencontre`;
- métadonnées discrètes;
- courte amorce;
- sections notionnelles avec emoji;
- exemple de prompt ou réponse IA;
- explication guidée;
- à essayer;
- récapitulatif;
- pour aller plus loin;
- références pédagogiques repliées.

## Boîtes IA

Tous les prompts, réponses simulées, extraits de sortie IA et résumés de modifications doivent utiliser une boîte IA.

Classes principales :

```text
aiBox
aiBoxPrompt
aiBoxResponse
aiBoxChanges
```

## Règles de maintenance

- Chaque contenu doit garder un identifiant stable.
- Le palier I doit rester utile seul.
- Une page publique doit être lisible pour l’étudiant avant d’être utile au suivi administratif.
- Les références pédagogiques doivent rester disponibles, mais discrètes.
- Les prompts et réponses IA doivent utiliser les boîtes IA réutilisables.
