# Codex IDE Extension

> Source: https://developers.openai.com/codex/ide (redirects to https://learn.chatgpt.com/docs/codex/ide)

Integrate Codex into your editor to maintain context and streamline development.

## Overview

Codex operates alongside your code, allowing you to incorporate active files and selections directly into prompts. This approach enables you to examine modifications in situ and delegate extensive tasks while preserving your momentum.

> "Work with Codex beside your code. Bring open files and selections into the prompt..."

[Install the extension](https://marketplace.visualstudio.com/items?itemName=openai.chatgpt)

---

## Key Features

### 01 Use Active Context

Reference open files and recent chats directly within the composer. Codex leverages the code currently visible to reduce the need for repetitive context setting.

> "Codex starts with the code you are already looking at, so you spend less time restating the problem."

### 02 In-Place Change Review

Analyze a concise summary and focused diffs directly beside the source code. This allows you to retain desired changes while keeping the rationale visible, without navigating away.

> "Read the summary, inspect a focused diff, and follow up in the same chat. Keep only the changes you want..."

### 03 Delegate Larger Tasks

Handle quick iterations locally, or connect to Codex web for more complex operations. You can return to the editor to review the results from the cloud environment.

> "Keep quick iterations local, or connect Codex web when a task needs more time and room."

## Quickstart Guide

1. **Install or Enable:** Select your IDE. VS Code and compatible editors use the Codex extension, while Xcode and JetBrains have specific integrations.
2. **Open Codex:**
   - **VS Code/Cursor/Windsurf:** Click the Codex icon or run "Codex: Open Codex Sidebar" in the Command Palette.
   - **Xcode:** Use the coding assistant and select Codex.
   - **JetBrains:** Open AI Chat and choose Codex.
3. **Start Chatting:** Open a project and request code explanations, modifications, or debugging help. Utilize Git checkpoints to manage changes.

---

## Capabilities in the IDE

- **Use Open Context:** Attach open files or selections to the composer to guide Codex's explanations or edits.
- **Review Beside Code:** View summaries and changed lines without opening new navigation panes, allowing for immediate acceptance or follow-up.
- **Delegate Work:** Choose local processing for rapid iteration or use Codex web to offload longer tasks, keeping the chat accessible for later review.

---

## Ideal Use Cases

- **Focused Edits:** Keep relevant files and Codex visible simultaneously.
- **Learning Code:** Inquire about symbols and files already open in the editor.
- **In-Place Review:** Inspect and apply edits alongside the source code.
- **Large Task Delegation:** Initiate cloud work from the IDE and return for results.
