## Règle de lecture

- Le `MIC` donne l’identité stable du contenu.
- La page publique donne la version lisible par l’étudiant.
- La position « Rencontre 1, 2 ou 3 » appartient au parcours actuel, pas au contenu.
- Les trois notions sont transversales.
- GitHub Copilot est l’exemple principal actuel, mais n’est pas la nature de ces modules.

## Rôle des modules pilotes

Ces modules permettent de vérifier :

- si le format de micro-module est assez léger;
- si les paliers I / II / III sont utiles;
- si les exercices sont suffisamment concrets;
- si les compétences observables sont bien liées aux modules;
- si les pages publiques restent provider-agnostiques lorsque la notion est transversale;
- si les modules peuvent servir de modèle pour produire les autres micro-modules du catalogue.

## Prochaine étape

Utiliser ces trois pilotes comme référence de qualité avant de produire les autres contenus par petits lots.
