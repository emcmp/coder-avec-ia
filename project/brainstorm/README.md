# Brainstorm

> Identifiant : `[DOC-BRN-001]`  
> Version : 0.2  
> Dernière modification : 2026-07-11  
> Statut : espace d’idées actif

Ce document sert de zone de dépôt libre pour les idées liées au parcours **Coder avec l’IA**.

Il n’est pas nécessaire que les idées soient complètes, classées, validées ou rédigées dans le bon format. L’objectif est de conserver les intuitions avant qu’elles disparaissent.

## Règle d’usage

Déposer ici :

- des idées de contenus;
- des exemples à utiliser plus tard;
- des outils à explorer;
- des frictions vécues avec l’IA;
- des activités possibles;
- des angles pédagogiques;
- des questions ouvertes;
- des risques ou limites à traiter;
- des idées de démonstrations;
- des liens ou sources à analyser plus tard.

Une idée peut être floue. Elle pourra être reformulée, classée, reliée à un module ou supprimée plus tard.

## À ne pas faire ici

- Ne pas chercher à tout structurer immédiatement.
- Ne pas transformer chaque idée en module.
- Ne pas bloquer une idée parce qu’elle n’est pas encore mûre.
- Ne pas utiliser ce document comme catalogue officiel.

## Idées déjà discutées

### Format pédagogique des séances

- Commencer une séance par une mauvaise manière de faire : prompt flou, contexte surchargé, consigne ambiguë, confiance trop rapide dans l’IA.
- Montrer ensuite pourquoi ça échoue ou pourquoi c’est fragile.
- Introduire la solution théorique juste assez pour comprendre.
- Terminer par une expérimentation courte.

### Cours orienté programmation

- Le parcours vise surtout les étudiants en programmation et les enseignants d’informatique.
- L’objectif n’est pas de faire un cours général sur l’IA ni un cours de réseaux neuronaux.
- L’angle principal : mieux utiliser l’IA dans un vrai travail de développement.

### Copilot comme outil de référence

- Utiliser GitHub Copilot comme outil principal pour les exemples.
- Ajouter des équivalences Codex et Claude Code seulement quand elles aident à transférer la notion.
- Éviter les longues comparaisons de produits.

### Modules et notions à couvrir

- Contexte utile pour l’IA.
- Spécification légère avant implémentation.
- Cartographie de codebase.
- Validation d’une réponse IA plausible.
- Agents CLI et workflows agentiques.
- Git, branches, diffs, issues et pull requests.
- RAG, recherche documentaire et sources.
- Skills, instructions globales, `AGENTS.md`, `CLAUDE.md`, `copilot-instructions.md`.
- Observabilité : traces, logs, contexte, usage, coûts.
- Sécurité, permissions, confidentialité et données sensibles.
- Modèles locaux : quand c’est utile, quand c’est une distraction.

### Idées de démonstrations

- Donner un prompt volontairement flou et comparer avec un prompt contextualisé.
- Demander à l’IA de modifier un codebase sans cartographie, puis refaire avec cartographie.
- Montrer une réponse IA plausible mais fausse ou incomplète.
- Comparer une tâche faite dans Copilot, Codex et Claude Code pour illustrer les différences de workflow.
- Montrer comment lire une diff avant d’accepter une suggestion.
- Montrer une tâche agentique trop large, puis la réduire.

### Idées de compétences

- Fournir un contexte utile à un assistant IA.
- Identifier où intervenir dans un codebase existant.
- Valider une réponse ou une production IA.
- Spécifier une tâche avant de l’implémenter avec l’IA.
- Intégrer l’IA dans un flux Git révisable.
- Encadrer les risques, permissions et données sensibles.

### Idées visuelles

- Utiliser une matrice icône-item pour les pages de synthèse.
- Garder une identité “Superhero pédagogique”.
- Utiliser des icônes par domaine : contexte, cartographie, validation, sécurité, agents, pédagogie.
- Éviter les longues listes plates quand une matrice visuelle peut aider.

### Idées de veille

- Créer un workflow de veille IA pour alimenter le cours.
- Surveiller YouTube, blogs, GitHub et documentation officielle.
- Toujours garder les sources et citations.
- Distinguer veille pour construire le cours et contenu directement enseignable.
- Regarder les tendances, popularité et signaux d’adoption.

- Applications et connecteurs ChatGPT, notamment le connecteur GitHub.
- MyGPT et possibilité de le brancher à un service web.
- Obsidian comme mémoire externe ou « second brain ».
- Agents CLI et combinaison CLI + skills pour construire ou livrer des agents.
